End-to-End System Workflow / தொடக்கம் முதல் முடிவு வரை

1. Prerequisites Setup / கணினித் தேவைகள்

English Explanation

Install Node.js (v18+) and Python (v3.12) compilers. A local MongoDB community service or cloud MongoDB Atlas cluster must be configured. Ensure Python is added to the system PATH environment variable.

தமிழ் விளக்கம்

கணினியில் Node.js (v18+) மற்றும் Python (v3.12) நிறுவப்பட வேண்டும். உள்ளூர் கணினியில் MongoDB சேவையைத் தொடங்கவும் அல்லது MongoDB Atlas கிளவுட் இணைப்பைத் தயார் செய்து கொள்ளவும்.

2

2. Configure Environment Keys / சர்வர் அமைப்புகள்

English Explanation

Configure the connection parameters inside server/.env. The backend Node.js server automatically handles cloud database connection failures and diverts queries to the local offline JSON fallback database.

தமிழ் விளக்கம்

server/.env கோப்பில் தரவுத்தள இணைப்பை அமைக்கவும். ஆன்லைன் கிளவுட் சர்வர் துண்டிக்கப்பட்டால், Node.js சர்வர் தானாகவே லோக்கல் கோப்பில் (`local_db.json`) இணைந்து கொள்ளும்.

server/.env 
PORT=5000
MONGO_URI=mongodb+srv://Ajith_db:Ajith%402004@ajithdev.0qvlvp8.mongodb.net/churn_data?appName=AJITHDEV
JWT_SECRET=super_secure_random_key_for_production_auth_98765
ML_SERVICE_URL=http://localhost:8000/predict

Power BI Integration & DAX / பவர் பிஐ பகுப்பாய்வு

English Explanation

Load the generated dataset/telecom_customer_churn.csv file into Microsoft Power BI Desktop. Create these three DAX measures in your model to compute business metrics:

தமிழ் விளக்கம்

உருவாக்கப்பட்ட dataset/telecom_customer_churn.csv கோப்பை Power BI Desktop மென்பொருளில் ஏற்றி, கீழே உள்ள DAX கணிப்பீட்டுச் சூத்திரங்களை அமைக்கவும்:

DAX Measure: Churn Rate (%) 
ChurnRate = DIVIDE(
    CALCULATE(COUNT(telecom_customer_churn[customerId]), telecom_customer_churn[churn] = "Yes"), 
    COUNT(telecom_customer_churn[customerId]), 
    0
) * 100
DAX Measure: Monthly Recurring Revenue (MRR) 
MRR = SUM(telecom_customer_churn[monthlyCharges])
DAX Measure: Revenue Lost (Monthly) 
Monthly Revenue Lost = CALCULATE(
    SUM(telecom_customer_churn[monthlyCharges]), 
    telecom_customer_churn[churn] = "Yes"
)

High Availability Failover / அவசரகாலத் தற்காப்பு

English Explanation

Fault-Tolerant Operations

• **Database Outage**: If MongoDB cloud goes offline, the server diverts operations to the local JSON file server/uploads/local_db.json.
• **ML Outage**: If the Python service goes offline, the server routes prediction queries to a JavaScript heuristic scoring engine.

தமிழ் விளக்கம்

தடையில்லாச் செயல்பாடு

• **தரவுத்தள செயலிழப்பு**: MongoDB முடங்கினால், தளம் தானாகவே உள்ளூர் local_db.json கோப்பைத் தொடர்பு கொண்டு தரவுகளைச் சேமிக்கும்.
• **AI சர்வர் செயலிழப்பு**: Python ML சர்வர் முடங்கினால், Node.js-க்குள் உள்ள மாற்று ஜாவாஸ்கிரிப்ட் விதிமுறை கணிப்பான் (Heuristic Engine) இயங்கி தொடர்ந்து கணிப்புகளைத் தரும்.
